# Brands

Use **Brands** to run multiple storefronts from a single UddoktaPay dashboard.  
Each brand has its own settings (logo, mail, notifications, themes, API keys, etc.) and can be mapped to one or more domains.

---

## Plan limits & upgrades

Your plan defines how many brands and **panel (dashboard) domains** you can use.

- **Basic**: 1 panel domain and **1 brand**.
- Need more brands or panel domains? Upgrade anytime from **my.uddoktapay.com**.

::: tip
If your license includes **multiple panel domains**, you can access the **same dashboard** from each of those domains.
:::

---

## How UddoktaPay decides “which brand” to use

When a request comes in, UddoktaPay resolves the active brand in this order:

1. **URL brand slug (if present)**  
   If the route includes a brand slug (e.g., `/default/api/checkout`), that brand is used.

2. **Domain match (if attached)**  
   If the host matches a domain attached to a brand, that brand is used (no slug needed).

3. **Default brand**  
   If neither a slug nor a matching domain is found, the **default brand** is used.  
   If no default brand is configured, a 404 will be returned.

Once resolved, links generated by the system will **automatically target the current brand**, so you don’t have to manually add the brand slug to every URL.

::: warning
If you remove a brand’s only domain and you haven’t set a default brand, requests without a slug may result in **404** until you add a domain or set a default brand.
:::

## Brands & domains

- **Brand domains** are public-facing domains that resolve a brand (e.g., `example.com`, `newdomain.com`).  
- A brand may have **zero, one, or many** domains.

**With a brand domain**  
- Routes live at the root of that domain:  
  `https://brand-domain.tld/api/checkout` (no slug)

**Without a brand domain**  
- The brand is available under its **slug** on your main domain:  
  `https://your-domain.tld/{brand_slug}/api/checkout`

::: warning
All brand and panel domains must point to the **same document root** as your panel.
:::

---

## Panel domains (reference)

“Panel domains” are the domains you use to access the dashboard (e.g., `pay.example.com`).  
If your package allows extra panel domains (or you purchase an upgrade), you can access the same dashboard from each of those domains.  
They should point to the **same document root** as your panel.

---

## Create a brand

1. Go to **Administration → Brands → Create Brand**.  
2. Enter **Name** and **Slug**.  
3. Click **Create**.

> The slug is used for the fallback URL when no domain is attached to the brand.

---

## Attach domains to a brand

1. Open **Administration → Brands → Edit** for the brand.  
2. In **Domains**, click **New domain** and enter the domain (e.g., `example.com`).  
3. Save changes.

**DNS & hosting**

- Point the domain (A/AAAA/CNAME) to the server hosting your panel.  
- Use the **same document root** as the panel.  
- Ensure **SSL** is installed for each domain.

When DNS/SSL is ready, requests to that domain will automatically resolve to the attached brand (no slug required).

## Examples

### Two brands on separate domains
- **Default** brand → `example.com`  
- **New Brand** → `newdomain.com`

Results:
- `https://example.com/api/checkout` → **Default** brand  
- `https://newdomain.com/api/checkout` → **New Brand**

### Brand without a domain
- **Default** brand (no domain attached)

Result:
- `https://your-domain.tld/default/api/checkout` (uses brand **slug**)

### Two domains on the **same** brand
- **Default** brand → `example.com`, `example.org`

Results:
- `https://example.com/api/checkout` and `https://example.org/api/checkout` both serve **Default** brand

::: tip
To connect **two domains** to the **same brand**, add both domains to that brand and point them to the **same document root**.
:::

---

## Best practices

- Use a **separate domain per brand** for cleaner URLs and SEO.  
- Keep slugs short and readable; they appear in fallback URLs.  
- Ensure **SSL** is enabled for every attached domain.  
- Assign a **default brand** to avoid accidental 404s when using the main domain without slugs.  
- When adding multiple domains to one brand, confirm all point to the **same document root**.

---

## Troubleshooting

- **Still seeing the slug path**  
  Add a domain to the brand and wait for DNS/SSL to propagate.

- **Domain shows panel login, not brand routes**  
  Verify the domain is attached to the correct brand and shares the panel’s document root.

- **Unexpected brand resolved**  
  Check the domain is attached to the intended brand; otherwise the **default brand** may be used.

- **404 after removing a brand domain**  
  Set a **default brand** or access via the brand slug until you attach a new domain.

- **Can’t create a brand**  
  You reached your plan limit. Upgrade from **my.uddoktapay.com**.
